May 29 arXiv cs.AI featured 25+ new papers, with Agent and LLM reasoning dominating. 为何重要：AutoSci 代表了 AI Agent 从"单任务执行"走向"跨项目科学研究全生命周期"的关键一步——记忆持久化、DAG 工作流、反馈进化三点结合，距离"AI 科学家"更进一步。SLAT 的 50% 推理长度缩减直接击中当前大模型推理成本痛点，具有即时工程价值。

DecomposeR + AdaCoM：DAG 规划强化学习与 Agent 上下文管理双突破
DecomposeR + AdaCoM: DAG Planning RL & Agent Context Management Twin Breakthroughs
(1) DecomposeR（多伦多大学 / Vector Institute）将研究计划表示为带类型的有向无环图（DAG），分两阶段训练——规划器 RL（学习图结构与查询分解）和回答器 RL（分支级执行与综合），在长文本基准上超越可比开源基线 5.1-8.0 分；(2) AdaCoM（蒙特利尔 / 华为诺亚）训练外部 LLM 管理冻结 Agent 上下文，首次揭示"保真度-可靠度"折衷：强 Agent 需要高保真上下文中存，弱 Agent 则需要更激进压缩。
(1) DecomposeR (U of Toronto/Vector Institute) represents research plans as typed DAGs and trains in two stages — planner RL (graph structure + query decomposition) and answerer RL (branch execution + synthesis), outperforming comparable open baselines by 5.1-8.0 points on long-form benchmarks; (2) AdaCoM (Montreal/Huawei Noah) trains an external LLM to manage frozen agent context, revealing the first "Fidelity-Reliability Trade-off": stronger agents benefit from higher-fidelity context preservation, while weaker agents need more aggressive compression.
为何重要：DecomposeR 的 DAG 规划范式与 Anthropic Dynamic Workflows 异曲同工——两者都在结构化子任务图而非线性链。AdaCoM 发现的"反向上下文需求"挑战直觉：弱模型需要更激进信息压缩而非更多信息，这对 Agent 系统设计有直接指导意义。
Why it matters: DecomposeR's DAG planning paradigm parallels Anthropic's Dynamic Workflows — both structure work as task graphs rather than linear chains. AdaCoM's counterintuitive finding — weaker models need more aggressive compression, not more context — has direct design implications for Agent systems.

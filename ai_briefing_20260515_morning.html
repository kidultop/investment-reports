OpenAI 详解 Codex Windows 沙箱设计：双账户隔离 + DPAPI 凭证保护
OpenAI Details Codex Windows Sandbox: Dual-Account Isolation + DPAPI Credential Protection
5月13日，OpenAI 发布了 Codex Windows 沙箱的详细技术说明。新设计使用两个本地用户账户运行：CodexSandBoxOffline（默认无出站网络）和 CodexSandBoxOnline，在系统层面隔离网络访问。在最终子进程启动前，完成 DPAPI 凭证保护、防火墙状态检查、codex-command-runner.exe 命令中转四层执行路径校验。2026 年 3 月 Windows 版 Codex 首次推出时缺少沙箱机制，5月 8 日公布的新设计优化了沙箱严格度，同时不影响常规开发效率。企业管理员可通过集中策略管理沙箱权限，满足大规模部署需求。
On May 13, OpenAI published detailed technical documentation for the Codex Windows sandbox. The new design uses two local user accounts: CodexSandBoxOffline (no outbound network by default) and CodexSandBoxOnline, isolating network access at the system level. Before final subprocess launch, four layers of enforcement are applied: DPAPI credential protection, firewall status check, codex-command-runner.exe command handoff. The March 2026 Windows Codex release initially shipped without sandboxing; the May 8 update optimized strictness without impacting regular development efficiency. Enterprise admins can manage sandbox permissions via centralized policies for large-scale deployments.
